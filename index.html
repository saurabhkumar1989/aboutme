Projects
Social media mining to locate traffic conditions
August 2017 - Current
The primary objective of this project is to triangulate road traffic incidents using social media data. Once geo-location data of the traffic incidents extracted, build spatial data layer over from the over Google map to display all of these incidents.

Motivation

Loads of data is generated on a daily basis on social platforms in the form of congestion notifications, slow moving traffic updates, traffic images, and news ticker data among others. These valuable pieces of information can be of great value to traffic administrators and commuters in proactive monitoring, decision making and effective analysis of the traffic condition in that particular area. With effective and proactive traffic administration and navigation as the primary focus, this project will tackle these problems by introducing a real-time system that takes the advantages of the social media data and combines with existing Geographical Information System like Google maps for real-time detection and diagnosis of traffic incidents.

Project Detail

Four major component for this projet are:

  • Data Extraction

    I tried to look for the existing freely available social media posts database for this project but I had no luck. So, I decided to go ahead to gather social media data by my own ( in my case only Twitter - for now). The data collector program was developed using Python( it was integrated into the Windows services ) to gathered tweets for a period of six months. For more visit git page here

  • Classifier Design

    The dataset contained tweets related to politics, personal communications, product promotions, traffic related incidents and much more. To filter out the tweets linked with traffic-related incidents, machine learning based SVM classifier was used. It consist of three major four major parts :

    1. Text Preprocessing Tweets contains substantial noise in it, like URLs, emoticons. In order to remove these unwanted entities from the text, regular expression (regex) used.
    2. Text Feature Extraction To convert a tweet into a feature vector, previously i had used the Bag of Words (BOW) method. The BOW method generates a term frequency model, which is basically a list of numbers produced on the basis of how many times a term is appearing in the text but problem BOW ignore the semantics of the words i.e. BOW not able to capture the relation between words. It treats every word as an individual identity. To overcome this, skip ngram based word2vec model used for feature extraction.
    3. Classifier Algorithm Different supervised classifier algorithms tried over the extracted data and found that the classifiers with complex decision had low accuracy. As an example Neural Network (with two layers) - 54%, Support Vector Machine (SWM) with Gaussian kernel - 94%, SVM with polynomial (of degree two) kernel - 93 %, Logistic Regression - 94 %. for more result click here - detail data. The main cause of this was evil of over-fitting as when the data was plotted, I figured out that some classifiers were overfitting over training data.

  • Geo parsing location information.

    Note - I am still working on this part

    This is the most stressful part of the entire project. Earlier I thought to detect every part of the location information (like a street name, places and building name etc.) but now I am narrowing down it to streets and places ( In future, I will expand it for other location relation information also). To find out the street names I am taking a hybrid approach i.e. I am using name entity recognition for buildings and gazetteer for toponym detection.

Key Point

  • Using deep learning skip gram based feature extraction method from text.

  • Created text classifier models using Neural Network, Support Vector Machine (with Gaussian and polynomial kernel) and accuracy of 93 %.

  • Geo parsing location information using Name Entity recognition for buildings and gazetteer for toponym detection.

Web Application to Visualize Traffic Incidents
January 2017 - May 2017
Designed web application architecture to visualize the road traffic incidents, using Google maps using MongoDb and MySQL.

System Details

The overall system is composed of...

Application Tier

This layer composed of Data Extractor, Web Framework and Inverted Index builder

  • Data Extractor

    Python script hits the twitter rest API's and stores the data in the MySQL server hosted on Amazon EC2.

  • Inverted Index Builder

    Python program to build index data structure in MongoDB that searches for and identifies items in a database correspond to keywords or characters specified by the user.

  • Web framework

    django framework to server all the HTTP request.

Client Tier

A responsive graphical user interface powered by Bootstrap, jquery and HTML5 for application to users to interact with the data.

Enterprise Tier

This layer contains MongoDB database having inverted indexes for search and MySQL database for the tweet data like tweet text, user information etc.

  • Databses: MySQL and MongoDB

  • Web Technologies: Django, jQuery and Bootstrap 3
