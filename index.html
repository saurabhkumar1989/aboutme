Text Classifier

August 2014 - December 2016

The main of this project is to build a classifier that can segment out the twitter post from the road traffic incidents from another post. In this project, the bag of words technique is used for text feature extraction and Neural Network algorithm as a classifier. Show Less

System Description

The overall system is composed of four parts.

  • The first section involved data collection using Twitter Rest API.

  • In the second stage, the task of noise removal was performed. In this step, certain parts of the tweets will not provide any relevant information like URL (Universal Resource Locator), emoticons and special characters like hashtags (#), at the rate (@), hyphen, etc. which are to be removed from the tweets.

  • In the third step, the filtered text is converted into a vector by using Bag of Words (BOW).

  • In final stage the newly generated feature vector is fed to a supervised classifier algorithm i.e. Neural Network Algorithm

Outcome

Though with the smaller data set the above classifier was working fine but with the bigger data set, system performance was poor and the reason behind it the feature extraction algorithm as Bag of Word ignore the semantics of sentences. And that motivates me to find some better feature extraction algorithms and that search landed me to deep learning methods.

System Details

The overall system is composed of...

Application Tier

This layer composed of Data Extractor, Web Framework and Inverted Index builder

  • Data Extractor

    Python script hits the twitter rest API's and stores the data in the MySQL server hosted on Amazon EC2.

  • Inverted Index Builder

    Python program to build index data structure in MongoDB that searches for and identifies items in a database correspond to keywords or characters specified by the user.

  • Web framework

    django framework to server all the HTTP request.

Client Tier

A responsive graphical user interface powered by Bootstrap, jquery and HTML5 for application to users to interact with the data.

Enterprise Tier

This layer contains MongoDB database having inverted indexes for search and MySQL database for the tweet data like tweet text, user information etc.
