Social media mining to locate traffic conditions

Motivation

Loads of data is generated on a daily basis on social platforms in the form of congestion notifications, slow moving traffic updates, traffic images, and news ticker data among others. These valuable pieces of information can be of great value to traffic administrators and commuters in proactive monitoring, decision making and effective analysis of the traffic condition in that particular area. With effective and proactive traffic administration and navigation as the primary focus, this project trying to tackle these problems by introducing a real-time system that takes the advantages of the social media data and combines with existing Geographical Information System like Google maps for real-time detection and diagnosis of traffic incidents.

Introduction

The primary objective of this project is to triangulate road traffic incidents using social media data. Once traffic incidents geo-location data extracted, a spatial data layer built over Google map to display all of these incidents.

Project Detail

Major component are:

  1. Data Extraction

    2. I tried to look for the existing freely available social media posts database for this project but I had no luck. So, I decided to go ahead to gather social media data by my own ( in my case only Twitter - for now). The data collector program was developed using Python( it was integrated into the Windows services ) to gathered tweets for a period of six months. For more visit git page here.

  2. Classifier Design

    3. The dataset contained tweets related to politics, personal communications, product promotions, traffic related incidents and much more. To filter out the tweets linked with traffic-related incidents, machine learning based SVM classifier was used. It consist of three major parts :

    • Text Preprocessing

      • Tweets contains substantial noise in it, like URLs, emoticons. In order to remove these unwanted entities from the text, regular expression (regex) used.

    • Text Feature Extraction

      • To convert a tweet into a feature vector, previously i had used the Bag of Words (BOW) method. The BOW method generates a term frequency model, which is basically a list of numbers produced on the basis of how many times a term is appearing in the text but problem BOW ignore the semantics of the words i.e. BOW not able to capture the relation between words. It treats every word as an individual identity. To overcome this, skip ngram based word2vec model used for feature extraction.

    • Classifier Algorithm

      • Different supervised classifier algorithms tried over the extracted data and found that the classifiers with complex decision had low accuracy. As an example Neural Network (with two layers) - 54%, Support Vector Machine (SWM) with Gaussian kernel - 94%, SVM with polynomial (of degree two) kernel - 93 %, Logistic Regression - 94 %. for more result click here - detail data. . The main cause of this was over-fitting as when the data was plotted, I figured out that some classifiers were overfitting over training data.

    Key Points

    • Deep learning Word2Vec method for feature extraction.

    • Created text classifier models using Neural Network, Support Vector Machine (with Gaussian and polynomial kernel) and accuracy of 93 %.

    • Geo parsing location information using Name Entity recognition for buildings and gazetteer for toponym detection.
